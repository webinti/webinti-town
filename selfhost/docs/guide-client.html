1. Avant de commencer

Bienvenue ! Ce guide vous accompagne pas à pas pour installer Webinti Town — votre bureau virtuel — sur votre propre serveur. Aucune compétence de développement n'est requise : uniquement un accès administrateur à un serveur Linux et à votre zone DNS. L'essentiel du temps d'installation est de l'attente (DNS, compilation, certificats).

Ce que vous installez

Le bundle déploie automatiquement quatre services sur votre serveur, orchestrés par Docker :

┌────────────── Caddy (HTTPS automatique) ─────────────┐ navigateur ─────►│ / → l'application (le bureau virtuel) │ │ /api /socket.io → serveur de jeu temps réel │ │ /pb → base de données et comptes │ └────────────────────────────────────────────────────────┘ navigateur ────── audio/vidéo ──────────► LiveKit (voir section 2)

Le HTTPS est entièrement automatique : les certificats sont obtenus et renouvelés sans aucune intervention de votre part.

Prérequis

ÉlémentDétail
Un serveur (VPS) Linux (Ubuntu 22.04+ ou Debian 12 recommandé), 2 vCPU / 4 Go RAM / 40 Go disque minimum, avec une adresse IP publique. Chez n'importe quel hébergeur (OVH, Hetzner, Scaleway…).
Docker + Docker Compose Installés sur le serveur : curl -fsSL https://get.docker.com | sh (le plugin compose est inclus).
Un nom de domaine Un sous-domaine dédié, ex. town.votre-entreprise.com, avec un enregistrement DNS A pointant vers l'IP du serveur. Obligatoire : sans HTTPS, les navigateurs bloquent caméra et micro.
Ports ouverts 80/tcp et 443/tcp (pare-feu du serveur et de l'hébergeur).
Compte LiveKit Cloud Gratuit, pour l'audio/vidéo (mode recommandé, voir section 2) : https://cloud.livekit.io.
Votre clé de licence Fournie avec votre abonnement Enterprise (format LIC-XXXX-YYYY). Si vous ne l'avez pas reçue, contactez Webinti.

2. Choisir le mode audio/vidéo

Les conversations audio/vidéo de proximité passent par un serveur média (LiveKit). Deux options :

Mode 1 — LiveKit Cloud (recommandé)Mode 2 — Auto-hébergé (avancé)
PrincipeL'audio/vidéo transite par le cloud LiveKit. Toutes les autres données (comptes, messages, documents) restent sur votre serveur.Tout reste sur votre serveur, y compris les flux audio/vidéo.
À faireCréer un projet sur cloud.livekit.io, copier 3 valeurs.2ᵉ sous-domaine + DNS, ouvrir 7881/tcp & 7882/udp, exploiter le serveur média.
CoûtGratuit jusqu'à un usage soutenu, puis à l'usage.Inclus dans votre serveur (prévoir plus de CPU et de bande passante).
Choisissez-le siVous voulez la simplicité (99 % des cas).Votre politique interne exige que rien ne sorte de votre infrastructure.
Mode 1 — préparer LiveKit Cloud (2 minutes) Créez un compte sur https://cloud.livekit.io → nouveau projet → dans Settings → Keys, notez : l'URL du projet (wss://votre-projet.livekit.cloud), l'API Key et l'API Secret. L'installateur vous les demandera.

3. Installation express (recommandée)

1Transférer et extraire le bundle sur le serveur
scp webinti-town-selfhost.tar.gz votre-user@IP-DU-SERVEUR:~
ssh votre-user@IP-DU-SERVEUR
mkdir webinti-town && tar xzf webinti-town-selfhost.tar.gz -C webinti-town
cd webinti-town/selfhost
2Lancer l'installateur guidé
./install.sh

Il vérifie Docker, puis vous demande dans l'ordre :

QuestionQuoi répondre
Domaine de l'applicationVotre sous-domaine, ex. town.votre-entreprise.com (le DNS doit déjà pointer sur ce serveur).
Email du compte administrateurL'email de la personne qui administrera l'espace (outils d'administration en jeu).
Clé de licence WebintiVotre LIC-XXXX-YYYY.
URL du serveur de licenceLaisser la valeur par défaut proposée.
Mode audio/vidéo (1/2)1 = LiveKit Cloud (recommandé). Il demande alors l'URL wss://…, l'API Key et le Secret notés en section 2.
Lancer maintenant ?o — l'installateur exécute docker compose up -d --build.

Le mot de passe administrateur de la base de données est généré automatiquement et écrit dans le fichier .env — vous en aurez besoin à la section 5.

Le premier lancement compile l'application : 5 à 10 minutes selon le serveur. Les certificats HTTPS sont obtenus automatiquement (rien à faire).

4. Installation manuelle (alternative)

Si vous préférez tout contrôler :

cd webinti-town/selfhost
cp .env.example .env
nano .env        # remplir selon le tableau de l'annexe A
docker compose up -d --build

Pour le mode audio/vidéo auto-hébergé (mode 2) : décommentez le bloc « SFU LOCAL » du .env (il active CADDYFILE=Caddyfile.local et COMPOSE_PROFILES=local-livekit), créez le 2ᵉ DNS A et ouvrez 7881/tcp & 7882/udp.

5. Premier démarrage (une seule fois)

1Créer le compte administrateur de la base de données

Ouvrez https://VOTRE-DOMAINE/pb/_/ — l'interface vous propose de créer le premier compte administrateur. Utilisez l'email et le mot de passe présents dans votre .env (lignes POCKETBASE_ADMIN_EMAIL / POCKETBASE_ADMIN_PASSWORD).

2Initialiser les données de l'application
docker compose exec server node dist/pocketbase/migrate.js

Crée les structures nécessaires (utilisateurs, kanban, messages, postes de travail). À ne faire qu'une fois.

3Redémarrer le serveur
docker compose restart server

Permet au serveur de finaliser ses collections avec le compte administrateur que vous venez de créer. Avant ce redémarrage, les journaux peuvent afficher des avertissements d'authentification — c'est normal et sans conséquence.

4Créer votre compte « hôte » dans l'application

Ouvrez https://VOTRE-DOMAINE, créez un compte avec exactement l'email administrateur déclaré à l'installation. Ce compte devient l'hôte : il dispose des outils d'administration en jeu. Tous vos collaborateurs peuvent ensuite créer leur compte librement.

6. Vérifier que tout fonctionne

docker compose ps                    # tous les services « running »
docker compose logs server | tail    # doit afficher : licence: active
  • HTTPS : https://VOTRE-DOMAINE s'ouvre avec un cadenas valide.
  • Licence : le journal ci-dessus affiche licence: active.
  • Audio/vidéo : connectez-vous à deux (deux navigateurs), rapprochez vos avatars : la conversation audio/vidéo doit s'établir. Testez aussi le partage d'écran.
✅ C'est terminé Votre espace est en production. Partagez simplement l'adresse https://VOTRE-DOMAINE à vos équipes.

7. Mises à jour & sauvegardes

Mettre à jour (quand Webinti livre une nouvelle version)

cd webinti-town/selfhost
docker compose pull              # composants tiers
docker compose up -d --build     # reconstruit l'application

Sauvegarder (recommandé : quotidien)

Toutes vos données vivent dans un volume Docker :

docker run --rm -v webinti-selfhost_pb_data:/data -v $(pwd)/backups:/backup \
  alpine tar czf /backup/pocketbase-$(date +%F).tar.gz -C /data .

Ajoutez cette commande à une crontab et copiez le dossier backups/ hors du serveur. Sauvegardez aussi votre fichier .env (une seule fois, il ne change pas).

8. Dépannage

SymptômeCause probable & solution
Le site ne répond pas / erreur de certificat Le DNS ne pointe pas (encore) sur le serveur — vérifiez avec dig +short VOTRE-DOMAINE, attendez la propagation. Vérifiez que 80 et 443 sont ouverts. Journaux : docker compose logs caddy.
L'application s'affiche mais pas d'audio/vidéo Vérifiez les 3 valeurs LiveKit du .env (URL wss://, Key, Secret), puis docker compose up -d pour ré-appliquer. En mode auto-hébergé : vérifiez 7882/udp & 7881/tcp ouverts aussi chez l'hébergeur.
« Licence Webinti expirée ou absente » Vérifiez LICENSE_KEY dans le .env (bien la clé fournie par Webinti) et que le serveur a accès à Internet en sortie, puis docker compose restart server. Si le message persiste, contactez Webinti.
Connexion / création de compte impossible L'initialisation n'a pas été faite — rejouez l'étape 5-2 (initialisation), puis vérifiez docker compose logs pocketbase.
Le serveur redémarre en boucle docker compose logs server — le plus souvent une variable .env manquante ou mal orthographiée (comparez à .env.example).

Support : contact@webinti.com — joignez la sortie de docker compose ps et docker compose logs --tail 100 server.

9. Votre licence

Votre instance vérifie automatiquement votre licence Enterprise une fois par jour auprès de Webinti. C'est transparent et sans effet sur les performances. Trois choses à savoir :

  • Une coupure Internet ne vous pénalise jamais : votre instance garde une réserve de validité de 7 jours. Une panne ponctuelle (chez vous ou chez Webinti) n'a aucun effet.
  • Si votre abonnement s'interrompt, l'accès se réduit progressivement (d'abord un simple avertissement, puis une limitation du nombre de connexions, puis un écran de suspension). Vos données ne sont jamais supprimées.
  • Dès la régularisation, tout revient automatiquement sous 24 h — ou immédiatement avec docker compose restart server.

10. Checklist récapitulative

  • VPS Linux prêt (2 vCPU / 4 Go), Docker + Compose installés
  • DNS A town.mon-domaine.com → IP du serveur (propagé)
  • Ports 80 et 443 ouverts (serveur + hébergeur)
  • Projet LiveKit Cloud créé — URL wss, API Key, API Secret notés
  • Clé de licence LIC-… reçue de Webinti
  • Bundle extrait, ./install.sh déroulé sans erreur
  • Compte admin de la base créé (/pb/_/) + initialisation exécutée
  • Compte hôte créé dans l'application avec l'email administrateur
  • Journal licence: active + test audio/vidéo à deux réussi
  • Sauvegarde quotidienne programmée (section 7)

Annexe B — Commandes utiles

# État des services

# Journaux en direct (serveur de jeu / proxy / base)

docker compose logs -f pocketbase
# Redémarrer un service

docker compose up -d --build
