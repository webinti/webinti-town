1. Comment marche le système

Le modèle commercial. L'auto-hébergement de Webinti Town est réservé à l'offre Enterprise (350 €/mois). Le client paie d'abord (Stripe, sur live.webinti.com), puis reçoit le bundle d'installation et une clé de licence. Stripe n'est jamais installé chez le client : il te sert uniquement à vendre.

Le kill-switch. L'instance du client appelle une fois par jour ton serveur de licence (licences.webinti.com). Si l'abonnement est actif, elle reçoit un jeton signé (Ed25519) valable 7 jours et fonctionne normalement. S'il cesse de payer, les jetons ne sont plus émis et l'instance se dégrade progressivement (jamais brutalement, jamais destructivement), puis se réactive automatiquement dès la régularisation.

Trois garanties
  • Un client qui paie n'est jamais coupé — même si ton serveur de licence tombe, le dernier jeton reste valable jusqu'à 7 jours.
  • Les données du client ne sont jamais détruites — l'expiration bloque l'accès, elle n'efface rien.
  • Un jeton ne peut pas être falsifié — seule ta clé privée (jamais livrée) peut en produire.
Lucidité sur la protection Le code tourne chez le client : un développeur déterminé pourrait patcher la vérification. La protection réelle = technique + contrat Enterprise. Le mécanisme rend le contournement délibéré et évident, donc juridiquement indéfendable — largement suffisant pour du B2B à 350 €/mois.

La chaîne complète (automatique)

1. Client paie Enterprise sur live.webinti.com (checkout tagué selfhost=true) 2. Stripe ─────── webhook ──────► licences.webinti.com/v1/stripe/webhook 3. Le serveur de licence CRÉE automatiquement la licence (clé générée, 100 users, active) 4. Tu la vois dans la console ─────► licences.webinti.com/admin 5. Tu livres au client : le bundle + sa clé de licence Chez le client : instance ── heartbeat 1×/j ──► licences.webinti.com ── jeton signé 7 j ──► Impayé → Stripe le signale → licence 'canceled' → l'instance se dégrade toute seule.

3. Vendre & livrer un client

1Le client souscrit → la licence se crée toute seule

Dès le paiement Enterprise validé sur live.webinti.com, la licence apparaît dans la console (statut active, clé auto-générée, 100 utilisateurs).

2Récupérer la clé dans la console

Ouvre https://licences.webinti.com/admin, repère la ligne du client (par son email), copie sa clé LIC-…. (Besoin ponctuel de créer une licence à la main — vente hors Stripe, test, geste commercial ? Le bouton « Créer » de la console le fait.)

4. La console d'administration

URL : https://licences.webinti.com/admin

Connexion : avec le LICENSE_ADMIN_TOKEN (ton mot de passe d'admin). Si tu ne l'as pas sous la main, récupère-le sur le VPS :

ssh openclaw@147.93.89.151 'grep LICENSE_ADMIN_TOKEN ~/webinti-license/license.env'

Ne pas confondre avec la clé private-key.pem (qui signe les licences, jamais utilisée pour se connecter).

Ce que tu y fais

  • Services état en direct de webinti-license, webinti-server, livekit-server, pocketbase (pastille verte = actif).
  • Licences tableau de toutes les licences (clé, client, plafond, statut, lien Stripe).
  • Créer une licence manuellement (clé auto-générée ou saisie).
  • Éditer client / nombre max d'utilisateurs.
  • Suspendre / Réactiver en un clic.
  • Supprimer une licence.

La console est le moyen principal de gestion. Les mêmes actions existent en API (annexe A) pour l'automatisation.

5. Cycle de vie des licences

Impayé — ce qui se passe automatiquement

Chronologie pire cas après l'échec de paiement (le client garde jusqu'à 7 jours de jeton déjà émis, puis les périodes de grâce s'enchaînent) :

MomentÉtatCe que vivent les utilisateurs du client
Échec de paiement (J0)activeRien — le dernier jeton (≤ 7 j) reste valable.
Expiration du jeton (≤ J+7)grace (3 j)Tout fonctionne normalement.
+3 j après expirationrestrictedMaximum 2 personnes connectées simultanément.
+7 j après expiration (≤ J+14)expiredÉcran « Accès suspendu » — aucune connexion, données intactes.
RégularisationactiveRetour automatique sous 24 h (ou immédiat si le client redémarre son conteneur).

Suspension / réactivation manuelle

Depuis la console (§4) : boutons « Suspendre » / « Activer » sur la ligne de la licence (litige, geste commercial, fin de contrat…). L'instance du client se dégradera à l'expiration de son jeton, ou reviendra sous 24 h après réactivation.

6. Opérations serveur (VPS)

Le serveur de licence vit dans ~/webinti-license/ sur le VPS (openclaw@147.93.89.151) : server.mjs, admin.html, sign.mjs, stripe-verify.mjs, keys/private-key.pem, license.env (secrets), licenses.json (registre — à inclure dans les sauvegardes).

# État & logs
sudo systemctl status webinti-license
journalctl -u webinti-license -f

# Redémarrer (sans mot de passe — règle NOPASSWD en place)
ssh openclaw@147.93.89.151 'sudo -n systemctl restart webinti-license'

# Mettre à jour le code (ex. server.mjs / admin.html)
scp server.mjs admin.html openclaw@147.93.89.151:~/webinti-license/
ssh openclaw@147.93.89.151 'sudo -n systemctl restart webinti-license'

Le certificat TLS de licences.webinti.com se renouvelle automatiquement (certbot). Le registre des licences (licenses.json) et license.env (qui contient le secret webhook + le token admin) doivent figurer dans tes sauvegardes du VPS.

Annexe A — API du serveur de licence

La console couvre tout le quotidien ; l'API reste utile pour scripter.

EndpointRôle
GET /healthVérification de vie (public).
GET /adminLa console web (page HTML).
GET /admin/status 🔒État des services + nombre de licences.
GET /admin/licenses 🔒Liste toutes les licences et leur statut.
POST /admin/license 🔒Crée / modifie une licence (merge — ne perd pas les champs Stripe).
POST /admin/license/:clé/cancel · /activate 🔒Suspend / réactive.
DELETE /admin/license/:clé 🔒Supprime une licence.
POST /v1/activateAppelé par les instances clientes (1×/jour) : renvoie un jeton signé 7 j si actif.
POST /v1/stripe/webhookReçoit les événements Stripe (signature vérifiée) : crée/active/annule les licences selfhost=true.

🔒 = en-tête authorization: Bearer <LICENSE_ADMIN_TOKEN>. États d'une instance : active → grace (3 j) → restricted (2 connexions) → expired.
